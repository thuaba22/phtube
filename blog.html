Exploring JavaScript & REST API

1. Discuss the scope of var, let, and const

In JavaScript, the scope of variables defined with var, let, and const differs. var has function scope, which means it is accessible throughout the entire function in which it is declared. let and const have block scope, limiting their accessibility to the block in which they are defined. Additionally, variables declared with const cannot be reassigned after their initial value is set, while let allows for reassignment.



2. Tell us the use cases of null and undefined

null and undefined are both used to represent missing or empty values in JavaScript. null is an intentional absence of any object value and is often used to indicate that a variable should have no value. undefined is typically the value assigned to a variable that has been declared but not yet initialized. It can also be used to represent the absence of a property in an object. Understanding when to use null or undefined depends on the specific use case and the desired semantics.



3. What do you mean by REST API?

An API, or application programming interface, is a set of rules that define how applications or devices can connect to and communicate with each other. A REST API is an API that conforms to the design principles of the REST, or representational state transfer architectural style.
